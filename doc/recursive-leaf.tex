\subsection{Implementation}
\label{sec:rec-leaf-impl}

The \textbf{leaf recursive} strategy implements the Mandelbrot computation through a
divide-and-conquer quadtree recursion.
Starting from the full $\mathit{ROWS}\times\mathit{COLS}$ matrix, each call to
\texttt{mandel\_tiled\_rec} inspects the four borders of its sub-tile:

\begin{itemize}
    \item If all border pixels share the same iteration count (\texttt{equal = 1}),
          the entire sub-tile is filled with that value via a \textbf{leaf task}.
    \item If the sub-tile is at or below the minimum tile size (\texttt{NCols <= TILE}),
          all interior pixels are computed individually via a \textbf{leaf task}.
    \item Otherwise the sub-tile is split into four quadrants (or two halves when only
          columns remain to be halved) and \texttt{mandel\_tiled\_rec} is called
          recursively for each quadrant -- \emph{sequentially}, with no task at the
          recursive call sites.
\end{itemize}

Tasks are therefore created only at the \emph{leaves} of the recursion tree.

The complete implementation (leaf tasks for fill and compute, sequential recursion at internal nodes) is in \texttt{codis/mandel-omp-rec(1).c}.

Because the recursive calls are sequential, only one thread walks the tree and generates
leaf tasks; all other threads execute those tasks.
Data races are avoided: each leaf task writes an exclusive region of \texttt{M}.
Shared histogram updates use \texttt{\#pragma~omp~atomic}.

\paragraph{Limitation.}
With the leaf strategy, task creation is inherently \emph{serial} -- only the thread
executing the single region can recurse and spawn.
This means that at the start of execution, while the tree is being traversed, most
threads remain idle waiting for tasks.
The Improved Leaf version (Section~\ref{sec:rec-improved}) addresses this bottleneck.
